# 进阶介绍

如果你已经跑通过第一章，可以从这篇文档理解项目为什么会有这么多模块，以及这些模块如何一起服务一件事：帮助你把一本长篇小说持续写下去。

## 一条主线：从想法到章节

项目的核心不是“多放几个 AI 生成按钮”，而是把长篇创作拆成一条可继续的路线：

1. 先确认这本书要给谁看、卖点是什么。
2. 准备世界规则、核心角色和主要冲突。
3. 把故事拆成卷、节奏段和章节任务。
4. 按章节任务生成正文。
5. 对正文做审核、修复和状态回填。
6. 把新的角色状态、伏笔和资料继续带到后续章节。

你可以把它理解为一个创作工作台：AI 负责帮你规划、生成和检查，系统负责保存结果、显示进度和提供继续入口。

## 自动导演适合解决什么

自动导演适合在你“不知道下一步该干什么”时使用。它会把开书过程拆成多个阶段，并在关键节点给出候选方案。

它更适合回答这些问题：

- 这个点子能写成什么类型的书？
- 哪个方向更适合继续展开？
- 需要哪些角色和世界规则？
- 当前是否已经可以开始写章节？
- 如果任务中断，应该从哪里继续？

## 创作中枢适合解决什么

创作中枢是你和系统沟通创作意图的入口。它适合用来解释当前进度、发起创作任务、查看系统建议，或者把一个模糊想法变成可以执行的下一步。

如果你不确定某个功能入口在哪里，可以先从创作中枢或新手上路开始。

## 资料为什么要进知识库

长篇小说很容易因为资料分散而失去一致性。知识库用于保存参考资料、设定、拆书结果和补充文本。后续生成章节时，系统可以根据当前任务召回相关资料，而不是只依赖你临时写在提示词里的几句话。

适合放进知识库的内容包括：

- 世界设定和规则说明。
- 参考作品分析。
- 角色资料和历史事件。
- 写作风格样本。
- 你希望后续章节遵守的资料。

## 写法引擎适合解决什么

写法引擎用于保存“这本书应该怎么写”的语言和叙事偏好。你可以从样本文本中提取写法特征，也可以手动调整规则，再把它用于试写、章节生成和后续修正。

它适合在这些场景使用：

- 想让后续章节保持某种叙述味道。
- 想减少模板化、AI 味过重的表达。
- 想把自己喜欢的样本文本整理成可复用写法。

## 模块之间怎么配合

推荐理解方式是：

- 创作类模块负责推进一本书。
- 资产类模块负责准备可复用资料。
- 系统类模块负责配置模型、提示词和运行偏好。

当你只想写书时，优先关注创作类模块；当你想提升稳定性和一致性时，再逐步使用资产类模块；当模型效果或成本需要调整时，再进入系统类模块。

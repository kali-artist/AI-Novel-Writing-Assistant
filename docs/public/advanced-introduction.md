# 进阶介绍

AI 小说创作工作台的核心价值，不是多做几个写作按钮，而是把长篇小说创作当成一条可以持续运行的 AI 生产链来设计。

## AI Native 长篇生产链

传统写作助手通常是“输入 Prompt -> 得到一段文本”。这个项目采用的是更接近生产系统的思路：

1. 用户给出灵感或目标。
2. AI 先帮助整理方向、卖点和读者感受。
3. 系统沉淀世界、角色、卷规划、章节任务等资产。
4. 章节执行链路按任务单生成正文。
5. 审核、修复、状态同步和知识回灌继续影响后续章节。

这意味着 AI 不只是生成正文的模型，而是参与规划、判断、调度和修复的系统角色。

## 自动导演

自动导演负责把一本书从模糊想法推进到可开写状态。它会处理：

- 开书方向候选。
- 书名与定位。
- 故事宏观规划。
- 本书世界准备。
- 角色准备。
- 卷战略与拆章。
- 章节执行衔接。

自动导演的重点是降低新手的前置认知负担。用户不需要先懂完整小说结构，也可以让系统给出明确下一步。

## Creative Hub

Creative Hub 是创作中枢。它承载对话、追问、规划、工具调用和任务状态，不只是普通聊天页。

它的定位是帮助用户理解当前进度、找到下一步动作，并把创作意图转成可执行任务。真正的长任务执行仍由后端任务、自动导演和章节链路承接。

## RAG 与知识回灌

项目支持知识库和拆书结果进入后续创作上下文。用户可以：

- 上传资料并建立知识索引。
- 对参考作品做结构化拆书。
- 把拆书结论发布到知识库。
- 在规划、续写和正文生成时召回相关资料。

近期的知识库索引已经支持并发 Embedding、Qdrant 并行写入、分块 facets 和内容 hash 去重，使大文档索引更快、更稳定。

## 写法引擎

写法引擎把“风格”从一次性提示词变成可保存、编辑、绑定和复用的长期资产。

用户可以从现有文本中提取写法特征，保存原文样本，并在后续试写、修正和整本绑定中复用这些规则。

## 角色与形象演变

拆书角色档案支持不同生成深度。深入档案会结合 notes 和原文 RAG 证据，按外形、性格、能力、关系、弧线、语言风格、价值观等维度组织内容。

角色形象演变会按章节扫描外貌、服装、状态和场景锚点。新发现的外貌信息会先进入待确认词条，用户勾选后再融合回角色外貌，避免系统自动污染角色档案。

## 桌面版与本地运行

项目提供 Windows 桌面版，适合不想手动配置 Node、pnpm 和开发环境的用户直接使用。

开发者也可以从源码运行。当前技术栈主要包括：

- React + Vite 前端。
- Express + Prisma 服务端。
- SQLite 默认本地数据库。
- Qdrant 用于完整知识库 / RAG 体验。
- LangChain / LangGraph 相关能力用于 AI 工作流探索。

## 为什么这类系统复杂

长篇小说不是单段文本生成。系统需要同时处理：

- 长期目标和章节目标。
- 世界规则和角色状态。
- 伏笔、承诺、节奏和读者期待。
- 模型输出的不确定性。
- 用户手动修改后的状态同步。
- 失败恢复、预算、任务排队和重试。

因此，这个项目的演进重点不只是“让 AI 写得更长”，而是让生产链更稳定、更可恢复、更容易被新手理解。

# 创作中枢

创作中枢是你和 AI 小说创作工作台沟通创作目标的主要入口。它适合把模糊想法变成可执行任务，也适合在写作中途询问下一步。

## 它能做什么

创作中枢可以帮助你：

- 整理一句灵感。
- 发起开书方向生成。
- 解释当前小说状态。
- 引导进入自动导演或章节任务。
- 根据已有世界、角色、知识库资料提出建议。
- 把你的自然语言目标转换成系统可执行任务。

它不是普通闲聊页，重点是服务小说完成流程。

## 能力边界

适合交给创作中枢的任务：

- “我有一个灵感，帮我判断适合写什么方向。”
- “这本书下一步应该做什么？”
- “主角设定有点弱，帮我补强。”
- “我想继续写下一章，入口在哪里？”
- “章节失败后我应该重试还是重新规划？”

不适合的用法：

- 长时间脱离小说项目的泛聊。
- 要求系统忽略已有角色、世界和章节状态。
- 用一句命令覆盖自动导演形成的结构。
- 把所有设定都写在聊天里，不保存到小说资产。

长期可复用的信息应进入小说资料、知识库、角色库或写法引擎。

## 和自动导演的关系

创作中枢负责理解你的意图，自动导演负责把意图推进成阶段化任务。

可以这样理解：

- 你在创作中枢说明目标。
- 系统判断目标属于开书、补设定、写章节、修复问题还是查看状态。
- 需要流程推进时，创作中枢把你带到自动导演或相关任务。
- 自动导演记录阶段、候选、暂停原因和恢复入口。

如果创作中枢给出下一步建议，优先沿着建议进入对应模块，而不是反复在对话里要求它直接完成所有工作。

## 任务类型清单

常见任务包括：

- 开书方向：从灵感生成题材、卖点和候选方向。
- 世界准备：整理规则、舞台、势力和限制。
- 角色准备：生成核心角色、关系和冲突。
- 章节准备：把卷计划拆成章节任务。
- 正文执行：进入章节生成、审核和修复链。
- 状态解释：说明当前项目停在哪一步。
- 恢复建议：根据任务中心和导演跟进给出继续入口。

不同任务会消耗不同模型能力和时间。长任务可以在任务中心查看状态。

## 使用建议

提问时提供清晰目标，例如：

- “我要创建一本东方玄幻升级流，主角从边境小城开始。”
- “这本书有世界和主角，帮我进入章节准备。”
- “第一章审核有问题，帮我判断是否继续修复。”

如果你只输入“继续”，系统会尽量根据项目状态判断下一步；但明确目标能减少误解。

## 遇到卡住怎么办

先确认三件事：

1. 任务中心是否有运行中或失败任务。
2. 导演跟进是否等待确认。
3. 小说基础信息是否缺失。

创作中枢适合解释和发起任务，事实状态仍以任务中心、导演跟进和小说数据为准。

## 自然语言意图到阶段的映射

创作中枢会尽量把用户语言转成系统可执行的主链动作。下面是常见映射：

| 用户说法 | 系统理解 | 可能触发阶段 |
|---|---|---|
| “我有一个灵感，帮我开书” | 生成书级候选方向 | `candidate_seed_alignment`、`candidate_direction_batch` |
| “这几个方向都不满意” | 修订候选或生成下一批 | `refine_candidates`、`candidate_direction_batch` |
| “这个方向可以，但标题不行” | 重做标题候选 | `candidate_title_pack` |
| “就用这个方向继续” | 确认候选并创建小说 | `novel_create` |
| “帮我把这本书的主线定下来” | 生成故事宏观和书契约 | `story_macro`、`book_contract` |
| “世界观还不够清楚” | 准备本书世界 | `world_setup` |
| “角色太弱，重新做一版” | 重新生成或修订角色阵容 | `character_setup` |
| “继续拆章” | 进入卷战略或结构化拆章 | `volume_strategy`、`beat_sheet`、`chapter_list` |
| “写下一章” | 检查是否有章节任务并进入执行 | `chapter_batch_ready`、`chapter_execution` |
| “这章没过，怎么办” | 判断修复、质量债务或重规划 | `chapter_quality_review`、`quality_repair`、`replan_required` |
| “继续上次的任务” | 查找当前 task/checkpoint | `continue`、`resume_from_checkpoint` |

这些映射不是关键词规则。系统应基于 AI 结构化理解和当前小说状态判断，不应靠固定词表硬分支。

## 与自动导演主链的衔接边界

创作中枢可以发起和解释任务，但不应该绕过自动导演产物：

- 方向确认应进入候选确认，而不是只在聊天里说“就这个”。
- 角色设定应沉淀到角色资产，而不是只留在聊天记录。
- 世界规则应进入本书世界或知识库，而不是每章重复粘贴。
- 章节目标应进入章节任务单，而不是临时要求模型“按这个写”。
- 质量问题应进入审核、修复或质量债务，而不是只让聊天给建议。

如果你希望某条设定长期生效，要把它保存到小说资料、知识库、角色库、世界资产或写法引擎。

## 什么时候不该用创作中枢直接推进

这些情况优先去对应模块：

| 情况 | 推荐入口 | 原因 |
|---|---|---|
| 后台任务失败 | 任务中心 | 需要看真实错误和命令状态。 |
| 自动导演等待确认 | 导演跟进 | 需要处理 checkpoint。 |
| 角色候选需要合并 | 角色页 | 需要查看具体候选和资产。 |
| 章节正文需要局部编辑 | 章节页 | 需要看到正文和审核证据。 |
| 知识库不命中 | 知识库 | 需要看索引和召回测试。 |
| 模型表现异常 | 模型路由/系统设置 | 需要调整模型绑定。 |

## 提问模板

更容易触发正确主链的提问方式：

| 目标 | 示例 |
|---|---|
| 开书 | “我想写一本都市异能长篇，主角是失败创业者，帮我生成开书方向。” |
| 继续 | “这本书停在自动导演流程里，帮我判断下一步入口。” |
| 补角色 | “角色准备阶段暂停了，帮我判断哪些候选该确认。” |
| 拆章 | “书契约和角色都准备好了，帮我进入第一卷章节任务。” |
| 恢复 | “任务中心显示 stale，帮我判断应该恢复还是重试。” |
| 修复 | “第一章审核没过，帮我判断是轻修复、质量债务还是重规划。” |

创作中枢的价值在于降低入口选择成本，但主链事实仍以任务中心、导演跟进和小说资产为准。

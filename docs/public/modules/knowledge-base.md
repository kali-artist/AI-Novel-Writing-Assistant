# 知识库

知识库用于保存长期会影响创作的资料，例如设定文档、参考作品、拆书结论、角色资料、世界规则和写法样本。它的作用是让资料在后续规划、续写和正文生成中继续被使用，而不是每次都靠你临时复制粘贴。

## 知识库能帮你做什么

- 上传和管理创作资料。
- 保存拆书发布结果，让拆书经验进入后续写作。
- 查看资料索引进度，确认文档是否可以被检索使用。
- 测试某份资料是否能被召回。
- 归档不常用资料，并在需要时恢复。
- 为小说、世界观、写法和章节生成提供长期参考。

## 适合什么时候用

- 有设定、资料、参考文本或拆书结论，希望后续写作继续使用。
- 写长篇时，经常担心 AI 忘记前面的世界规则或资料背景。
- 想把多个参考来源统一整理，而不是散落在本地文件和聊天记录里。
- 想检查某份资料是否真的能被后续生成用上。
- 想让拆书结果、写法样本和世界资料成为可复用资产。

## 开始前准备

知识库是增强能力，不是开始写小说的前置条件。你可以先不配置知识库，直接创建小说和生成章节；当你开始需要长期资料、参考作品或复杂设定时，再启用知识库。

如果要完整使用资料检索能力，通常需要配置可用的 Embedding 模型和向量库连接。普通用户可以先按设置页提示完成基础配置，再回到知识库上传资料。

上传前建议先整理文本：

- 一份文档尽量只放同一类资料。
- 标题写清楚资料用途。
- 参考作品、世界设定、写法样本最好分开保存。
- 不再使用的资料可以归档，避免干扰后续召回。

## 推荐使用路径

1. 打开“知识库”。
2. 上传设定、参考文本或整理好的资料。
3. 查看索引状态，等待资料处理完成。
4. 使用召回测试确认资料能被找到。
5. 在小说、世界、拆书或写法流程中选择或绑定这份资料。
6. 不再使用时归档；需要重新使用时恢复并重建索引。

如果你从拆书模块发布结果，知识库会把它作为拆书发布资料保存，方便回到来源拆书继续查看。

## 如何进入后续创作链路

知识库资料会在多个地方发挥作用：

- 开书和规划时，提供题材、设定和参考作品信息。
- 世界和角色生成时，提供背景规则和人物资料。
- 章节写作时，提供当前任务相关的资料参考。
- 拆书发布后，让作品分析结论继续被后续创作召回。
- 写法引擎中，可以从知识库原文提取写法资产。

知识库不是正文编辑器。它更像长期资料仓库：把重要资料放进去，再让创作链路在需要时使用。

## 常见问题

### 不配置知识库能开始写小说吗

可以。知识库是增强能力，不是开书前置条件。先跑通创作流程，再逐步补资料，通常更适合新手。

### 上传后为什么还不能用

上传只是保存资料。资料还需要完成索引，才能在召回测试和创作链路中稳定使用。先查看索引状态和最近任务。

### 什么资料适合放进知识库

适合放长期会重复使用的内容，例如世界规则、势力资料、参考作品分析、拆书结论、角色背景、写法样本。临时灵感可以先放在小说项目里。

### 文档太多会不会干扰生成

会有可能。建议用清楚的标题、归档不再使用的资料，并在具体小说或任务里选择真正相关的资料。

### 归档会删除资料吗

归档不是删除。归档资料会退出默认使用范围，需要时可以恢复。恢复后通常要重新处理索引，才能再次被稳定召回。

### Qdrant 和 RAG 是必须理解的吗

不需要。你只要知道：配置完整后，知识库可以更好地按当前任务查找资料。技术细节交给系统和设置页处理。

# 系统设置

系统设置用于配置模型供应商、API Key、知识库连接、数据库相关偏好和基础运行选项。第一次使用时，系统设置是必须完成的入口。

## 优先配置什么

建议按这个顺序配置：

1. 模型供应商。
2. API Key。
3. Base URL。
4. 默认模型。
5. 连接测试。
6. 可选知识库设置。
7. 可选任务和界面偏好。

先确认模型可用，再进入新手上路。

## API Key 和供应商

填写 API Key 时注意：

- 不要多复制空格。
- 确认 Key 对应的供应商和 Base URL。
- 确认账户额度和模型权限。
- 如果供应商兼容 OpenAI 接口，按兼容接口填写。

连接失败时，先用供应商控制台或简单请求确认 Key 是否有效，再回到应用测试。

## Base URL 和模型名称

Base URL 需要和供应商文档一致。常见问题包括：

- 多写或少写路径。
- 使用了网页地址而不是 API 地址。
- 模型名称和供应商实际名称不一致。
- 代理或网络限制导致请求无法到达。

模型名称建议从供应商文档复制，不要自行猜测。

## Qdrant 设置

Qdrant 用于知识库向量检索。它不是开书和生成第一章的硬性前置条件，但会影响资料召回、拆书沉淀和长期一致性。

配置时确认：

- Qdrant 服务可访问。
- 地址和端口正确。
- 集合或命名策略符合应用配置。
- 索引任务能够完成。

知识库不命中时，不要只改提示词；先确认 Qdrant 连接和索引状态。

## SQLite 和本地数据

桌面版会保存本地数据库和配置。请把它当作重要数据处理：

- 不要随意删除数据库文件。
- 重要小说定期导出或备份。
- 迁移、重置或清理前先确认备份。
- 排查问题时保留日志和数据副本。

任何可能删除数据的操作都应先备份。

## 设置后的验证

完成设置后，建议执行一个最短验证：

1. 测试模型连接。
2. 创建测试小说。
3. 让自动导演生成方向候选。
4. 生成第一章。
5. 查看任务中心状态。

这条路径能同时验证模型、数据库、任务队列和主链入口。

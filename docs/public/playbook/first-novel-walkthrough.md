# 第一本小说实操路径

这是一条从空项目到第一批章节准备完成、并进入第一章执行的操作手册。每一步都标注对应自动导演阶段、产物位置和停下来时该看哪里。

## 路径概览

| 步骤 | 用户动作 | 对应阶段 | 产物保存位置 | 停下来先看 |
|---|---|---|---|---|
| 1 | 配置模型 | 系统配置 | 系统设置、模型路由 | 系统设置 |
| 2 | 输入灵感 | `candidate_seed_alignment` | 自动导演任务 | 任务中心 |
| 3 | 生成方向 | `candidate_direction_batch` | 候选方向批次 | 导演跟进 |
| 4 | 选择方向/标题 | `candidate_title_pack` | 候选方向、标题组 | 方向选择页 |
| 5 | 创建小说 | `novel_create` | 小说项目 | 小说列表 |
| 6 | 生成宏观故事和书契约 | `story_macro` / `book_contract` | 故事宏观、书契约 | 小说页、导演跟进 |
| 7 | 准备世界和角色 | `world_setup` / `character_setup` | 本书世界、角色候选 | 角色页、世界页 |
| 8 | 应用角色阵容 | `character_cast_apply` | 正式角色 | 角色库 |
| 9 | 生成卷规划 | `volume_strategy` / `volume_skeleton` | 卷战略、卷骨架 | 卷规划入口 |
| 10 | 拆章和细化 | `beat_sheet` / `chapter_list` / `chapter_detail_bundle` | 节奏板、章节清单、章节任务单 | 导演跟进、任务中心 |
| 11 | 进入章节执行 | `chapter_batch_ready` → `chapter_execution` | 正文草稿、审核报告 | 章节页、任务中心 |

## 第一步：配置模型

打开系统设置，配置默认模型。第一次使用只需要确保一个模型可用。

确认：

- API Key 有效。
- Base URL 正确。
- 默认模型能返回中文。
- 任务中心没有旧的失败任务干扰。

如果模型连接失败，先看《常见问题》和《故障排查》。

## 第二步：输入灵感

进入新手上路或创作中枢，输入一句灵感。不要手写完整大纲，让自动导演先生成方向。

示例：

> 一个新手作者进入灵感市场，寻找被遗忘的故事。

对应阶段：`candidate_seed_alignment`、`candidate_project_framing`。

产物：自动导演任务和候选生成输入。

## 第三步：生成方向批次

系统会进入 `candidate_direction_batch`，生成候选方向。你会看到不同定位、卖点、主角路径、冲突和推进循环。

选择时看：

- 哪个方向最容易持续多章。
- 主角目标是否清楚。
- 卖点是否适合目标读者。
- 冲突是否能不断升级。

如果都不满意，可以生成下一批或修订候选。

## 第四步：确认方向和标题

标题组属于 `candidate_title_pack`。标题不必最终定稿，但要能代表当前方向。

可选动作：

- 确认当前标题。
- 只重做标题。
- 定向修正候选。
- 回到上一批候选。

确认后，系统进入 `novel_create`。

## 第五步：创建小说

确认候选后，系统创建小说项目。创建成功后，你可以在小说列表看到新项目。

如果创建失败：

1. 打开任务中心。
2. 查看 `confirm_candidate` 命令错误。
3. 如果是临时错误，重试。
4. 如果是数据错误，保留错误信息再反馈。

## 第六步：生成故事宏观和书契约

自动导演会生成 `story_macro` 和 `book_contract`。

产物包括：

- 整本故事定位。
- 核心冲突。
- 推进循环。
- 目标读者。
- 前 30 章承诺。
- 叙述边界和硬约束。

查看位置：小说基础信息、故事宏观、导演跟进。

## 第七步：准备世界和角色

`world_setup` 会准备本书世界；`character_setup` 会生成角色候选。

角色可能触发 `character_setup_required`。这不是失败，而是要求你确认角色阵容。

处理方式：

- 确认合适角色。
- 合并重复候选。
- 退回重做角色阵容。
- 调整角色身份锚点。

不要在角色明显不合格时强行进入卷规划。

## 第八步：应用角色阵容

角色确认后，`character_cast_apply` 会把角色写入正式资产。此后章节执行会读取这些角色状态。

查看位置：基础角色库、本书角色页、导演跟进。

## 第九步：生成卷战略和卷骨架

`volume_strategy` 生成卷级路线，`volume_skeleton` 生成卷骨架。

你需要确认：

- 第一卷目标是否明确。
- 每卷承诺是否不同。
- 主角成长是否有阶梯。
- 伏笔和读者期待是否能持续。

如果卷规划不对，优先修卷战略，不要直接改后面的章节清单。

## 第十步：节奏板、章节清单和章节细化

结构化拆章包括：

- `beat_sheet`：生成卷内节奏节点。
- `chapter_list`：拆出章节清单。
- `chapter_sync`：同步章节执行合同。
- `chapter_detail_bundle`：细化章节任务单和执行资源。

这些阶段属于高内存任务，同一本书同范围通常不应重复启动多个任务。看到“已有自动导演任务正在处理同一范围”时，先去任务中心看正在运行的任务。

## 第十一步：进入章节执行

`chapter_batch_ready` 表示章节批次准备好。你可以：

- 手动确认进入章节执行。
- 授权自动执行章节。
- 暂停并检查章节任务。

第一章生成后，看正文、审核问题、修复结果和质量债务。正文可用但存在局部问题时，可以先继续后续章节，后面再处理质量债务。

## 常见停顿

| 停顿位置 | 说明 | 下一步 |
|---|---|---|
| `candidate_selection_required` | 等你选方向 | 选择、修订或重做候选。 |
| `character_setup_required` | 等你确认角色 | 确认/合并/重做角色。 |
| `volume_strategy_ready` | 卷战略完成 | 确认后进入拆章。 |
| `chapter_batch_ready` | 章节准备完成 | 进入章节执行。 |
| `replan_required` | 质量修复可能影响后续 | 查看原因，再决定重规划或继续。 |

## 完成标准

第一轮实操完成的标志：

- 小说项目存在。
- 书契约和宏观故事存在。
- 世界和角色资产可查看。
- 卷战略、节奏板和章节清单存在。
- 至少第一章有章节任务单。
- 第一章正文生成并进入审核/修复结果。

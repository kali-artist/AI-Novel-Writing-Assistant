# 使用方法

这是一条适合第一次使用的路线。目标不是一次弄懂所有功能，而是先创建一本书，跑通从灵感到第一章的流程。

## 第一步：配置模型

启动应用后，先进入系统设置或模型路由，配置至少一个可以调用的模型。

你通常需要准备：

- API Key。
- 模型名称。
- 如果使用兼容接口，可能还需要 API Base URL。

如果不确定怎么分配模型，先让规划、正文、审阅都使用同一个可用模型即可。等你跑通流程后，再按成本和效果拆分模型路由。

## 第二步：创建一本测试小说

进入新手上路或小说列表，创建一本测试小说。第一次不需要准备完整设定，只要给出一句灵感。

示例：

> 一个普通维修工发现城市地铁线路连接着被遗忘的地下王国。

系统会根据这句话生成可选择的方向，而不是要求你先写完整大纲。

## 第三步：选择开书方向

自动导演会生成不同方向的候选方案。你可以重点看三件事：

- 这套方向的主角和冲突是否吸引你。
- 读者期待是否清楚。
- 你是否愿意继续写这个故事几十章。

如果方向不满意，可以继续生成；如果某套方向接近你的想法，可以围绕它做局部调整。

## 第四步：推进世界、角色和拆章

选定方向后，系统会继续准备本书世界、角色阵容、卷规划和章节拆分。

这一阶段你可以先接受系统默认建议。后续如果发现角色不合适、世界规则太弱、章节节奏不对，再回到对应模块调整。

## 第五步：生成第一章

当章节任务准备好后，进入章节执行。这里会围绕当前章节任务生成正文，并提供审核、修复、保存和继续入口。

第一次建议只生成第一章。读完结果后，再决定是否继续推进后续章节。

## 第六步：逐步使用资产模块

跑通第一章后，可以按需要使用这些模块：

- 知识库：保存资料、设定、参考文本和拆书结论。
- 拆书：分析参考作品或自己的稿子。
- 写法引擎：保存叙事风格和语言偏好。
- 题材基底库、推进模式库：积累可复用的类型和爽点结构。
- 世界样本库、基础角色库：沉淀可复用世界与角色。

这些模块不必一次全部配置。它们的作用是让你后续写得更稳，而不是挡在第一章之前。

## 遇到失败时怎么处理

长链路任务可能遇到模型失败、额度不足、网络中断或内容质量不满意。遇到问题时，先查看任务中心和导演跟进。

优先尝试：

- 查看失败原因。
- 继续、恢复或重试任务。
- 切换模型后再试。
- 扩大预算或缩小生成范围。
- 对局部章节进行修复，而不是重建整本书。

系统通常会保留已经完成的内容。不要急着删除项目，先找恢复入口。

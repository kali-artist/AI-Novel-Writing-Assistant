# 使用方法

这份指南面向第一次使用 AI 小说创作工作台的用户，帮助你从下载安装到创建一本书，并理解主要功能入口。

## 1. 下载并启动

优先使用 Windows 桌面版：

1. 打开 [Latest Release](https://github.com/ExplosiveCoderflome/AI-Novel-Writing-Assistant/releases/latest)。
2. 下载 `Setup.exe` 安装版。
3. 安装后启动应用。

如果你不想安装，也可以下载 portable 版本，放到任意目录直接运行。

## 2. 配置模型

第一次使用前，需要在设置页配置可用模型。项目支持多种模型提供商，例如 OpenAI、DeepSeek、SiliconFlow、xAI 等。

通常需要准备：

- API Key。
- 模型名称。
- 如使用兼容接口，可能需要 API Base URL。

不同链路可以使用不同模型，例如规划、正文、审阅和图片生成可以分别配置。

## 3. 创建小说

进入小说创建入口后，你可以先输入一句灵感，例如：

> 一个普通维修工发现城市地铁线路其实连接着被遗忘的地下王国。

不需要一开始就写完整世界观、主线和人物。系统会帮助你把灵感扩展成可推进的整本方向。

## 4. 使用自动导演开书

自动导演会围绕你的灵感生成方向候选、标题组和推荐理由。你可以：

- 选择一套方向继续。
- 要求继续生成新方向。
- 针对某一套方向做定向重做。
- 让系统自动推进到可开写状态。

自动导演会逐步推进项目设定、故事宏观规划、本书世界、角色准备、卷战略、拆章和章节执行。

## 5. 准备世界和角色

项目支持“本书世界”概念。你可以：

- 从世界库导入世界样本。
- 根据本书主题生成世界。
- 自定义本书世界。

角色准备会结合世界规则、势力倾向和故事需求生成更贴合舞台的人物。角色不是孤立表单，而是会进入后续章节上下文。

## 6. 拆章并进入章节执行

当卷战略和节奏拆章准备好后，可以进入章节执行页。这里会看到：

- 当前章节任务单。
- 已保存正文。
- 执行计划。
- 正文生成、审核和修复动作。
- 状态同步和伏笔回填。

你可以逐章推进，也可以在合适时启动更长范围的批量生产。

## 7. 使用知识库和拆书

知识库用于保存资料、设定、参考文本和拆书结论。完整 RAG 体验需要配置 Qdrant。

拆书适合学习参考作品或诊断自己的稿子。系统会把作品拆成剧情、人物、世界观、写法等结构化信息，并支持发布到知识库，让后续创作链路检索使用。

## 8. 使用写法引擎

写法引擎可以从样本文本里提取风格特征。你可以保存这些特征，之后在试写、正文生成或整本绑定中复用。

它适合解决“我想让后续章节继续保持某种叙事味道”的问题。

## 9. 处理失败和继续

长篇生产链可能会遇到模型失败、预算不足、任务中断或质量问题。遇到这些情况时，优先查看：

- 当前任务状态。
- 自动导演或任务中心给出的下一步建议。
- 是否可以恢复、重试、扩容预算或局部修复。

不要急着删除项目或重建。系统通常会保留已经完成的产物，并提供继续推进的入口。

## 10. 推荐使用路径

第一次使用建议按这个顺序：

1. 配置模型。
2. 创建一本测试小说。
3. 输入一句灵感。
4. 用自动导演生成方向候选。
5. 选择方向并推进到可开写状态。
6. 查看本书世界、角色准备和拆章结果。
7. 进入章节执行生成第一章。
8. 根据结果尝试审核、修复、知识库和写法引擎。

先跑通一条完整链路，比一开始调很多高级参数更重要。
